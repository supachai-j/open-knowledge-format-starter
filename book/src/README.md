# คำนำ

ยินดีต้อนรับสู่ **คู่มือ Open Knowledge Format (OKF)** ฉบับภาษาไทย

หนังสือเล่มนี้จะพาคุณสร้างและดูแล **knowledge base ที่ AI ช่วยดูแล** ด้วยรูปแบบ
Open Knowledge Format — ฐานความรู้ที่เก็บเป็นไฟล์ Markdown ธรรมดากับ YAML frontmatter
ที่ทั้ง **คนและ AI agent อ่าน เขียน และใช้งานร่วมกันได้** โดยไม่ต้องพึ่ง SDK หรือฐานข้อมูลเฉพาะทาง

## หนังสือเล่มนี้เหมาะกับใคร

- **ผู้เริ่มต้น** ที่อยากมี knowledge base ส่วนตัว/ของทีม ที่ค้นได้และไม่ผูกกับ vendor
- **นักพัฒนา/ทีมข้อมูล** ที่อยากให้ AI agent เข้าถึงความรู้ภายในองค์กรอย่างเป็นระบบ
- **สถาปนิกระบบ/ทีม platform** ที่ต้องวางระบบใช้ร่วมกัน **ข้าม session และข้ามทีม ระดับองค์กร (on-prem)**

ไม่ต้องมีพื้นฐาน OKF มาก่อน ขอแค่พอใช้ command line และ Git ได้บ้าง

## คุณจะได้อะไร

เมื่ออ่านจบ คุณจะ:

- เข้าใจว่า OKF คืออะไร และต่างจาก RAG แบบเดิมอย่างไร
- สร้าง knowledge base แรกได้ภายในไม่กี่นาที
- เก็บความรู้ (ingest) ค้นหา (search) และดูแลให้ข้อมูลไม่เน่าเสีย
- ดู knowledge graph แบบ interactive แบบไฟล์เดียว (เปิด offline ได้)
- วางระบบใช้ร่วมกันระดับองค์กรแบบ self-host พร้อม MCP, hybrid search และการคุมการเขียนหลายแบบ

## แผนผังของหนังสือ

| ภาค | เนื้อหา | เหมาะกับ |
|-----|---------|----------|
| 1 | รู้จัก OKF | ทุกคน — เริ่มที่นี่ |
| 2 | เริ่มต้นใช้งาน (ติดตั้ง · สร้าง KB แรก) | ผู้เริ่มต้น |
| 3 | แนวคิดหลัก (concept · frontmatter · ลิงก์) | ผู้เริ่มต้น |
| 4 | การใช้งานประจำวัน (ingest · query · validate · viz) | ผู้ใช้ทั่วไป |
| 5 | เขียนให้ดี (best practices) | ผู้ใช้ทั่วไป |
| 6 | ระดับองค์กร (self-host · MCP · lease · security) | ทีม platform/SA |
| ผนวก | อ้างอิง CLI · FAQ · อภิธานศัพท์ | อ้างอิง |

## ข้อตกลงในหนังสือ

- คำสั่งที่ต้องพิมพ์ใน terminal จะอยู่ในกล่องโค้ด:
  ```bash
  python3 tools/okf-validate.py ./wiki
  ```
- **ศัพท์เทคนิค** เช่น `concept`, `frontmatter`, `bundle`, `MCP` คงไว้เป็นภาษาอังกฤษเพื่อให้ตรงกับเอกสารต้นทางและโค้ด
- กล่องสีเหลือง (blockquote) คือข้อควรระวังหรือ "เคล็ดลับ"

> **หมายเหตุเรื่องเวอร์ชัน:** OKF เป็นสเปก **v0.1** (เผยแพร่ 12 มิ.ย. 2026 โดย Google Cloud) ยังเป็นช่วงเริ่มต้น
> สเปกหลักบังคับแค่ฟิลด์ `type` เท่านั้น — "แนวปฏิบัติที่ดี" อื่น ๆ ในเล่มนี้ส่วนใหญ่มาจากชุมชน LLM-wiki และ
> reference implementation ของ Google ไม่ใช่ตัวสเปกเอง
